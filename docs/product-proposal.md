# Product Proposal — EventAI

## Обоснование идеи

Demo Day AI Talent Hub (ИТМО) — мероприятие на 330 проектов, 10 параллельных залов, ~500 участников. У задачи два независимых контура:

1. **Контур участника:** гость/эксперт/бизнес приходит на мероприятие и не может эффективно выбрать, что смотреть. При 6 залах видит максимум 1/6 программы (~35 проектов из 212 в день). Выбор становится случайным.

2. **Контур организатора:** один человек вручную кластеризует 330 проектов, распределяет экспертов, сводит расписание «за ночь до DD». ChatGPT кластеризует с дубликатами и потерями. 80 студентов выпали из программы из-за проблем с данными.

Предыдущее решение (бот Demo-Hero с Tinder-механикой) не масштабируется: после ~20 пар пользователь устаёт, механика не понимает контекст «зачем пришёл», организатору не помогает.

**Валидация:** 5 CustDev-интервью (организатор, эксперт, руководитель хаба, гость, инициатор проекта). Ключевой сигнал от организатора: «Если это можно упростить, это сильно снимет нагрузку».

---

## Цель и метрики

### Цель

Построить PoC агентной системы, которая:
- Для участников: через диалог в Telegram собирает профиль интересов и выдаёт персональную программу с объяснениями
- Для организаторов: автоматизирует кластеризацию проектов по залам, распределение экспертов, генерацию расписания

### Метрики успеха

**Продуктовые:**

| Метрика | Текущее (без системы) | Целевое (с PoC) |
|---|---|---|
| Время составления расписания | Дни (ручная работа) | Часы (LLM + ручная доводка) |
| Доля «потерянных» студентов | ~24% (80 из 330) | <5% (валидация при импорте) |
| Конверсия гостя: вход → программа | Нет данных | Измеряемо (телеметрия в боте) |

**Агентские (качество рекомендаций):**

| Метрика | Значение | Условия |
|---|---|---|
| NDCG@15 | 0.82 | 10 профилей × 330 проектов, ручная разметка 0-3 |
| Precision@15 | 0.71 | Sanity check, не научный результат |
| Recall@15 | 0.78 | Нет baseline, нет IAA |

**Технические:**

| Метрика | Целевое |
|---|---|
| Latency команд/кнопок (p95) | < 1 000 мс |
| Latency AI-операций (p95) | < 10 000 мс |
| Latency batch-кластеризации (330 проектов) | < 120 000 мс |
| Доступность в день DD (10:00-20:00) | 99.5% |
| Стоимость LLM на 200 гостей + орг.операции | ~$1-2 |

## Сценарии использования

### Основной сценарий: гость получает персональную программу

```
1. /start → выбор роли (студент / абитуриент / бизнес / другое)
2. Диалоговое профилирование (LLM, макс. 2 уточняющих вопроса)
3. Подтверждение профиля (теги, цели, контекст)
4. Генерация рекомендаций (embedding → Qdrant top-30 → rerank → LLM summaries → top-15)
5. Показ программы (must_visit top-8 + if_time bottom-7)
6. Agent mode: свободный диалог с tool-calling
   - show_project — карточка проекта
   - compare_projects — матрица сравнения 2-5 проектов
   - generate_questions — Q&A-подсказки
   - get_followup / get_pipeline — follow-up пакет / бизнес-пайплайн
   - rebuild_profile — перепрофилирование
```

### Сценарий: бизнес-партнёр

```
1. Профилирование: компания, должность, бизнес-цели (technology/hiring/investment/partnership)
2. Рекомендации с фильтром по business_objectives
3. Бизнес-пайплайн: interested → contacted → negotiating → closed_won/lost
4. Генерация LOI (Letter of Intent) по шаблону
```

### Сценарий: организатор кластеризует проекты

```
1. Импорт Excel → валидация → дедупликация (SHA-256 хеш файла)
2. Запуск AI-кластеризации (LLM генерирует JSON-распределение)
3. Валидатор проверяет hard-инварианты (полнота, уникальность, баланс)
4. Безопасные автоисправления (пропущенные → в наименьший зал, дубликаты → первое назначение)
5. UI: drag-n-drop правки (is_manual=True, аудит)
6. Подтверждение → фиксация в БД → основа для расписания и матчинга
```

### Edge-кейсы

| Кейс | Поведение |
|---|---|
| Размытый запрос («что-нибудь интересное») | Бот обязан задать уточняющий вопрос (ограничение в промпте). После 2 реплик — фиксирует что есть, выдаёт широкий топ |
| Проект без описания | Участвует в кластеризации по тегам, получает низкий score при рекомендациях |
| Дубликаты при импорте | SHA-256 хеш файла + проверка повторной загрузки в окне 10 минут |
| Дубликаты при кластеризации | Валидатор оставляет первое назначение, остальные убирает |
| LLM «потеряла» проекты при кластеризации | Автоисправление: добавляет в наименьший зал (source="autofix") |
| Все API-ключи в cooldown | Используется ключ с самым давним fail |
| LLM полностью недоступна | Профилирование: дефолтный вопрос без LLM. Рекомендации: tag overlap scoring. Кластеризация: fail с уведомлением |
| Многоязычность | Промпты на русском, LLM обрабатывает английский ввод (теги часто на EN). Специальной обработки нет |
| Троллинг/спам | Не обрабатывается. Rate limiting отсутствует (open issue) |
| Потеря сессии (перезапуск контейнера) | orphan_text_handler восстанавливает из БД по /start |

### Технические (SLO)

| Параметр | Значение |
|---|---|
| p95 latency: команды/кнопки | < 1 000 мс |
| p95 latency: CRUD | < 2 000 мс |
| p95 latency: AI-операции | < 10 000 мс |
| p95 latency: batch (кластеризация 330 проектов) | < 120 000 мс |

| RPO: подтверждения/оценки | 0 минут (нельзя потерять) |
| RPO: AI-контент (подборки, вопросы) | Допустима потеря (можно перегенерировать) |

### Операционные (бюджет)

| Параметр | Значение |
|---|---|
| LLM-провайдер | OpenRouter (GPT-5-Mini по умолчанию, $0.25/M in, $2.0/M out) |
| Стоимость на гостя | ~$0.004-0.005 |
| Стоимость на 200 гостей + орг. | ~$1-2 |
| Economy-режим | GPT-4o-Mini ($0.15/$0.60), DeepSeek ($0.14/$0.28) |
| Инфра | Yandex Cloud: 2 vCPU, 4 GB RAM, 30 GB SSD |
| Пиковая нагрузка | ~200 одновременных сессий, ~30 RPS |

### Регуляторные

- 152-ФЗ: данные на территории РФ (Yandex Cloud)
- Контакты только с согласия участника
- Все критические AI-решения модерируются организатором

**Компоненты:**
- **Telegram Bot (PTB 21.x):** ConversationHandler с 6 состояниями, PicklePersistence
- **Web Admin (React 19 + Vite):** TanStack Query, Tailwind, shadcn/ui
- **Core API (FastAPI):** 87 endpoints, SQLAlchemy 2.0, Alembic (34 миграции)
- **LLM Client:** OpenRouter API, multi-key rotation с cooldown, fallback model
- **Celery + RabbitMQ:** async processing для LLM-операций (9 task types)
- **PostgreSQL 16:** 35 таблиц, операционные данные + аудит
- **Qdrant:** векторная БД для semantic search по проектам
- **APScheduler:** 9 периодических задач (напоминания, брифинги, эскалации)
- **Traefik:** reverse proxy, SSL termination

## Data Flow: что делегируется LLM, что нет

### Делегируется LLM (через OpenRouter API)

| Операция | Вход | Выход | Fallback |
|---|---|---|---|
| Профилирование гостя | Текст пользователя + контекст роли | JSON: action=reply/profile, interests, goals | Дефолтный вопрос без LLM |
| Рекомендации: summaries | Top-15 проектов + профиль гостя | 2-3 предложения на проект | Первые 1-2 предложения description |
| Кластеризация проектов | JSON: проекты (id, title, tags, desc до 500 символов) + constraints | JSON: rooms[{room_index, name, theme_rationale, project_ids}] | Fail с уведомлением в UI |
| Adjacent tags (для матчинга) | Список всех тегов события | JSON: adjacency pairs | Только exact + fuzzy matching |
| Agent chat (tool-calling) | System prompt + chat history + tools | Text response или tool_call | Fallback model (GPT-4o-Mini) |
| Q&A-подсказки | Проект + профиль гостя | Список вопросов | Хардкодированные дефолтные вопросы |
| Матрица сравнения | 2-5 проектов + критерии | Таблица сравнения | «Не удалось сгенерировать» |

### НЕ делегируется LLM (детерминированная логика)

| Операция | Реализация |
|---|---|
| Embedding профиля | OpenRouter Embeddings API (google/gemini-embedding-001), не LLM chat |
| Qdrant similarity search | Косинусная близость, top-30 |
| Schedule-aware rerank | In-memory: room_bonus + conflict_penalty |
| Валидация кластеризации | Python: проверка полноты/уникальности/баланса |
| Автоисправления кластеризации | Python: missing → smallest room, duplicates → first assignment |
| Fuzzy matching экспертов | SequenceMatcher (порог 0.8) + IDF-веса |
| Импорт/дедупликация | SHA-256 хеш, проверка окна 10 минут |
| Расписание: слоты, drag-n-drop | Детерминированная генерация + UI |
| Рассылки, напоминания | APScheduler + шаблоны |
| Аутентификация | Telegram ID, whitelist организаторов |
