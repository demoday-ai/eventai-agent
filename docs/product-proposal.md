# Product Proposal - EventAI Agent

## Обоснование идеи

Demo Day AI Talent Hub (ИТМО) - 330 проектов, 10 залов, ~500 участников. При 6 залах гость видит ~35 из 212 проектов в день. Выбор случайный.

Предыдущее решение (бот Demo-Hero, механика свайпов) не масштабируется: после ~20 пар пользователь устает, контекст визита не учитывается.

MVP системы EventAI (бот + админка + бэкенд) разработан на AI Talent Camp 2026. **В рамках бонус-трека реализуется агентская часть** - Telegram-бот с диалоговым профилированием, рекомендательным пайплайном и инструментами. Остальные компоненты системы (веб-админка, инфраструктура) разработаны на кэмпе и используются агентом через API.

**Валидация:** 5 CustDev-интервью. Ключевой сигнал от организатора: "Если это можно упростить, это сильно снимет нагрузку".

---

## Цель и метрики

### Цель

PoC AI-агента в Telegram, который:
- Через диалог собирает профиль интересов гостя
- Выдает персональную программу с пояснениями
- Анализирует артефакты проектов (презентации, код) и отвечает на вопросы по содержимому
- Собирает доступность и оценки экспертов
- Маршрутизирует вопросы участников к организаторам

### Метрики успеха

**Продуктовые:**

| Метрика | Без системы | С PoC |
|---|---|---|
| Конверсия гостя: вход - программа | Нет данных | Измеряемо |
| Покрытие экспертами залов | Ручное | Автоматический матчинг |

**Агентские (качество рекомендаций):**

| Метрика | Значение | Условия |
|---|---|---|
| NDCG@15 | 0.82 | 10 профилей x 330 проектов, ручная разметка 0-3 |
| Precision@15 | 0.71 | Проверочный замер, не научный результат |
| Recall@15 | 0.78 | Нет бейзлайна, нет IAA |

**Технические:**

| Метрика | Целевое |
|---|---|
| Задержка команд/кнопок (p95) | < 1 с |
| Задержка AI-операций (p95) | < 10 с |
| Доступность в день DD (10:00-20:00) | 99.5% |
| Стоимость LLM на 200 гостей | ~$1-2 |

## Сценарии использования

### Гость получает персональную программу

```
1. /start - выбор роли (студент / абитуриент / бизнес / другое)
2. Диалоговое профилирование (LLM, макс. 2 уточняющих вопроса)
3. Подтверждение профиля (теги, цели, контекст)
4. Генерация рекомендаций (эмбеддинг - Qdrant top-30 - переранжирование - LLM-резюме - top-15)
5. Показ программы (обязательные 8 + по возможности 7)
6. Свободный диалог с инструментами агента:
   - show_project - карточка проекта (с данными из презентации и GitHub)
   - compare_projects - матрица сравнения 2-5 проектов
   - generate_questions - подсказки для Q&A по содержимому проекта
   - get_followup / get_pipeline - пакет контактов / воронка для бизнеса
   - rebuild_profile - перепрофилирование
   - ask_organizer - вопрос организатору
```

### Анализ артефактов проекта

```
1. При импорте данных (в админке) указываются ссылки на презентацию (PPTX/PDF) и/или GitHub
2. Агент при импорте парсит артефакты:
   - PPTX - python-pptx: текст из каждого слайда
   - PDF - pymupdf/pdfplumber: текст постранично
   - GitHub - git clone --depth 1: README + структура + ключевые файлы
3. Извлеченный текст сохраняется в БД (parsed_pitch, parsed_repo)
4. Агент при show_project / generate_questions / compare_projects
   использует полный контент в контексте
```

### Бизнес-партнер

```
1. Профилирование: компания, должность, цели (технологии/найм/инвестиции/партнерство)
2. Рекомендации с учетом бизнес-целей
3. Воронка: заинтересован - связались - переговоры - сделка/отказ
4. Генерация LOI по шаблону
5. Ответы агента по содержимому презентации/кода проекта
```

### Эксперт: слоты и оценки

```
1. Эксперт получает приглашение в боте
2. Выбор: конкретные слоты или все проекты сразу
3. Результат уходит в матчинг (с учетом выбранного времени)
4. Брифинг: зал, проекты, описания, подсказки для Q&A
5. После защиты: оценки по критериям в боте
6. Результаты на дашборде в админке
```

### Вопрос участника - ответ организатора

```
1. Участник пишет вопрос в боте
2. Бот пересылает в админку через API
3. Организатор отвечает
4. Ответ приходит участнику в бот
```

### Граничные случаи

| Случай | Поведение |
|---|---|
| Размытый запрос | Уточняющий вопрос. После 2 реплик - фиксация, широкий топ |
| Проект без описания | Низкий балл при рекомендациях |
| LLM полностью недоступна | Профилирование: вопрос без LLM. Рекомендации: пересечение тегов |
| Потеря сессии | Восстановление из БД по /start |

## Архитектура агента (в контексте системы)

```
                          +--------------------+
                          |    Web Admin       |
                          |    (React 19)      |
                          +--------+-----------+
                                   |
+---------------+         +--------v-----------+
| Telegram Bot  |-------->|  FastAPI (Core API) |
| (PTB 21.x)   |<--------|  87 эндпоинтов     |
| Агент:        |         +--+------+------+---+
| - профилир.   |            |      |      |
| - рекоменд.   |         +--v--+ +-v----+ +-v---+
| - инструменты |         |PgSQL| |Qdrant| |LLM  |
| - эксперты    |         |     | |(vec) | |Client|
| - вопросы     |         +-----+ +------+ +-----+
+---------------+
    ^  Scope бонус-трека
```

Агент реализован как ConversationHandler (python-telegram-bot 21.x) с 6 состояниями. Использует бэкенд-сервисы системы:

| Сервис | Что дает агенту |
|---|---|
| LLM Client (OpenRouter) | Профилирование, резюме, агентский диалог с инструментами |
| Qdrant | Векторный поиск проектов по эмбеддингу профиля |
| PostgreSQL | Хранение профилей, рекомендаций, оценок, вопросов |
| Celery + RabbitMQ | Асинхронная обработка LLM-операций |

## Разделение: LLM vs детерминированная логика

### Делегируется LLM

| Операция | Вход | Выход | Откат |
|---|---|---|---|
| Профилирование гостя | Текст + контекст роли | JSON: interests, goals | Вопрос без LLM |
| Резюме рекомендаций | Top-15 проектов + профиль | 2-3 предложения на проект | Первые предложения описания |
| Агентский диалог | Системный промпт + история + инструменты | Текст или вызов инструмента | Запасная модель (GPT-4o-Mini) |
| Анализ артефактов | parsed_pitch/parsed_repo + вопрос | Ответ по содержимому проекта | "Артефакт не загружен" |

### Не делегируется LLM

| Операция | Реализация |
|---|---|
| Эмбеддинг профиля | OpenRouter Embeddings API (Gemini) |
| Поиск по Qdrant | Косинусная близость, top-30 |
| Переранжирование | room_bonus + conflict_penalty |
| Парсинг PPTX/PDF | python-pptx / pymupdf - текст в БД |
| Парсинг GitHub | git clone --depth 1 - README + структура в БД |
| Сбор доступности экспертов | Бот: выбор слотов - БД - матчинг |
| Сбор оценок экспертов | Бот: оценки по критериям - БД - дашборд |
| Маршрутизация вопросов | Бот - API - админка - ответ - бот |
