# EventAI — AI-агент-куратор Demo Day

## Что за задача

На Demo Day магистратуры AI Talent Hub (ИТМО) — 330 проектов, 10 параллельных залов, 2 дня. Гость физически видит ~16% программы. Организатор сводит расписание вручную за ночь до события. Эксперты идут на защиты вслепую — без информации о проектах.

## Для кого

- **Внешние гости** (индустриальные партнёры, инвесторы, HR, менторы) — не могут найти релевантные проекты в потоке из сотен
- **Организаторы** (~5 чел.) — тонут в операционке: импорт данных, кластеризация, расписание, рассылки
- **Эксперты** (~50 чел.) — не получают информацию о проектах до защиты, не могут выбрать удобные слоты
- **Бизнес-партнёры** (~15-20 чел.) — не находят проекты под свою проблематику, follow-up хаотичный

## Какая боль сейчас

- Гость пропускает 84% программы из-за параллельности залов
- Организатор кластеризует 330 проектов вручную (или через ChatGPT с дубликатами и потерями)
- 80 студентов «выпали» из расписания из-за проблем с данными
- Предыдущий бот (Demo-Hero, Tinder-механика) не масштабируется: после ~20 свайпов пользователь устаёт
- Follow-up после мероприятия — хаотичный, без структуры

## Что делает PoC на демо

**Telegram-бот (агентская часть):**
- Диалоговое профилирование гостя (LLM, 2-3 реплики → структурированный профиль)
- Персональные рекомендации проектов (embedding → Qdrant → schedule-aware rerank → LLM summaries)
- Tool-calling агент: карточка проекта, сравнение 2-5 проектов в матрице, Q&A-подсказки, follow-up пакет
- Бизнес-пайплайн: статусы проектов (interested → contacted → negotiating → closed), генерация LOI
- Разные сценарии для 4 ролей (студент, абитуриент, бизнес, другое)
- Graceful degradation на каждом уровне (LLM, embeddings, Qdrant)

**Веб-админка (оргконтур):**
- Импорт данных из Excel + валидация + дедупликация
- AI-кластеризация проектов по залам (LLM → валидатор → ручные правки → аудит)
- Распределение экспертов (IDF-взвешенный fuzzy matching + LLM adjacent tags)
- Генерация расписания, drag-n-drop, экспорт .ics/.xlsx
- Рассылки, напоминания, брифинги экспертам
- Дашборд покрытия

## Что НЕ делает PoC (out-of-scope)

- **Организация встреч 1:1** — только запрос контакта автора, без бронирования слотов
- **Загрузка артефактов участниками** — проекты загружает организатор из Excel, нет self-service загрузки питч-деков/видео/репозиториев
- **Анализ питч-деков и GitHub-репо** — агент работает только с title/tags/description, не парсит презентации и код
- **Сбор доступности экспертов через бота** — организатор назначает, эксперт не выбирает слоты
- **Чат-поддержка бот↔админка** — нет канала «вопрос в боте → ответ из админки»
- **Обратная связь экспертов в боте** — оценки проектов не собираются через бота
- **E2E-тесты** — только unit/integration (317 тестов)
- **PII-фильтрация** — нет автоматической анонимизации перед отправкой в LLM
- **Rate limiting** на входе Telegram-бота
