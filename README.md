# EventAI — AI-агент-куратор Demo Day

## Что за задача

Demo Day магистратуры AI Talent Hub (ИТМО) — 330 проектов в 10 параллельных залах за 2 дня. Гость физически успевает на ~16% программы и выбирает почти вслепую. Организатор сводит расписание вручную за ночь до события. Эксперты приходят на защиты без информации о проектах.

EventAI — агентная система, которая закрывает оба контура: помогает участникам найти релевантное, а организаторам — подготовить и провести мероприятие.

## Для кого

- **Внешние гости** (партнёры, инвесторы, HR, менторы) — 190 регистраций, доходимость ~50. Не могут найти «свои» проекты среди сотен
- **Организаторы** (~5 чел.) — импорт, кластеризация, расписание, рассылки, контроль покрытия — всё вручную
- **Эксперты** (~50 чел.) — не получают информацию до защиты, не могут выбрать удобные слоты, оценки собираются в неудобных таблицах
- **Бизнес-партнёры** (~15-20 чел.) — не находят проекты под задачу, follow-up хаотичный

## Какая боль сейчас

- Гость при 6 залах видит ~35 из 212 проектов, остальные пропускает
- Организатор кластеризует через ChatGPT — дубликаты, потери, нестабильный формат
- 80 студентов выпали из расписания из-за кривых данных и ручного процесса
- Бот Demo-Hero (Tinder-механика) не масштабируется: после ~20 свайпов устают, контекст «зачем пришёл» не учитывается
- У эксперта нет способа сообщить своё доступное время, нет удобного способа ставить оценки
- Вопросы от участников летят в личку организатору — нет единого канала

## Что делает PoC на демо

### Telegram-бот (агентская часть)

- **Профилирование через диалог:** LLM уточняет роль/интересы/цели за 2-3 реплики, формирует структурированный профиль (JSONB: теги, ключевые слова, NL-summary, для бизнеса — компания/должность/цели)
- **Рекомендации:** embedding профиля (Gemini) → Qdrant cosine search (top-30) → schedule-aware rerank (бонус за зал, штраф за конфликт) → LLM-summaries для top-15
- **Анализ артефактов проектов:** парсинг питч-деков (PPTX/PDF — извлечение текста со всех слайдов) и GitHub-репозиториев (клон → README + структура + ключевые файлы). Агент отвечает на вопросы по реальному содержимому проекта, а не только по короткому описанию
- **Tool-calling агент:** 7+ инструментов — карточка проекта (с данными из презы/кода), сравнение 2-5 проектов в матрице, Q&A-подсказки, follow-up пакет, бизнес-пайплайн с LOI
- **Сбор доступности экспертов:** бот предлагает эксперту выбрать удобные слоты или получить все проекты сразу; результат уходит в матчинг
- **Сбор оценок экспертов:** после защиты эксперт ставит оценки проектам прямо в боте, результаты видны на дашборде в админке
- **Канал «бот → админка»:** участник задаёт вопрос в боте (перенос слота, уточнение зала) → организатор видит в админке → отвечает → ответ приходит в бот
- **Graceful degradation:** на каждом уровне — LLM fail → дефолтные вопросы, Qdrant fail → tag overlap, ключ заблокирован → ротация + fallback-модель

### Веб-админка (оргконтур)

- Импорт данных из Excel + валидация + дедупликация (SHA-256)
- AI-кластеризация по залам: LLM → валидатор (hard/soft инварианты) → автоисправления → drag-n-drop → аудит
- Распределение экспертов: IDF-взвешенный fuzzy matching (SequenceMatcher ≥0.8) + LLM adjacent tags
- Расписание: генерация слотов, drag-n-drop, экспорт .ics/.xlsx (с ФИО и описаниями)
- Рассылки, напоминания, брифинги, дашборд покрытия
- Входящие вопросы от участников из бота — ответы из админки

### Тестирование

- Unit/integration: 317 тестов (217 frontend Vitest + 100 backend pytest)
- E2E-тесты: сквозные сценарии (онбординг → профилирование → рекомендации → tool-calling)

## Что НЕ делает PoC (out-of-scope)

- **Организация встреч 1:1** — только запрос контакта автора, без бронирования слотов
- **Self-service загрузка артефактов студентами** — питч-деки и ссылки на GitHub загружает организатор при импорте, студенты не загружают сами
- **Автоматическая PII-фильтрация** — нет детектора персональных данных перед отправкой в LLM
- **Rate limiting** на входе Telegram-бота
- **Локальная/on-prem LLM** — всё через OpenRouter (данные покидают РФ при LLM-вызовах)
